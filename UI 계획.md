# Scene A 드로잉 UI 계획

> 작성일: 2026-04-29
> 상태: 계획 확정 (구현 미착수)

---

## 1. 컨셉

- **디제틱(diegetic) UI**: `Play Ref/초안.png` 참고. 도구가 화가 책상처럼 자연스럽게 놓여 있고, 물건을 클릭/조작해 기능을 사용한다.
- 추상적인 버튼 UI를 최소화하고, 세계관 안에 도구가 녹아드는 방향.

## 2. 전체 레이아웃

`초안.png` 그대로 따름.

```
[좌측 절반]                      [우측 절반]
  배경 + 손님                       작업대 (1인칭 시점)
  하단 대사창                       - 좌상단: Undo / Redo
                                   - 우상단: 시계 (PM 시간)
                                   - 좌측 중간: 팔레트 (8칸)
                                   - 중앙: 캔버스 (이젤+스케치북)
                                   - 우측 중간: 펜 홀더 (펜·붓·지우개)
                                   - 그림 밑 중앙: 굵기 슬라이더
                                   - 좌하단: SUBMIT
                                   - 우하단: RESET
```

## 3. 도구

| 도구 | 특징 |
|------|------|
| 펜 | 선명, 일정한 굵기 |
| 붓 | 부드러운 끝선, 살짝 투명도 있음 |
| 지우개 | 캔버스를 배경색으로 덮어씀 |

- 처음부터 세 도구 모두 제공
- 추가 도구(스프레이/마커/스머지 등)는 후 해금 예정

## 4. 굵기 슬라이더

- 위치: 그림 밑 중앙
- 범위: **2px ~ 24px**
- 슬라이더 옆에 **현재 굵기 미리보기 점** 표시
- 도구별로 굵기를 **각각 따로 기억** (펜 / 붓 / 지우개 독립)
- 디자인: 미확정

## 5. 팔레트

- **8칸 슬롯**, 처음부터 기본 8색으로 모두 채워둔다.
- 캐리커처 우선순위(얼굴 / 머리 / 선 / 그림자 / 볼터치) 기준 실용 구성.

| 슬롯 | 색 |
|------|------|
| 1 | 검정 |
| 2 | 흰색 |
| 3 | 밝은 살색 |
| 4 | 어두운 살색 |
| 5 | 진한 갈색 |
| 6 | 중간 갈색 |
| 7 | 회색 |
| 8 | 붉은 분홍 |

- **남색** 등 부수 색상은 기본 팔레트 제외 → 나중에 추가 색상 해금이나 슬롯 수정으로 사용
- 슬롯 클릭 = 색 선택
- 슬롯 **우클릭 / 길게 누르기** = RGB 피커 팝업 → 슬롯 색을 직접 수정 가능
- 더블클릭, 별도 수정 버튼은 사용하지 않음

## 6. RGB 컬러피커

- 슬롯 우클릭/길게 누르기 시 떠오르는 **팝업**
- 디자인 톤: **나무 테두리 + 베이지 배경** (대화창과 동일 계열)
- **RGB 수치만** 표시 (HSL 제외)
- **스포이드 내장**
  - 스포이드 클릭 → 캔버스 색 찍기 모드 전환
  - 캔버스에서 픽셀을 찍으면 **현재 수정 중인 슬롯 색**이 그 색으로 바뀜
  - 별도 외부 스포이드 도구 두지 않음

## 7. Undo / Redo

- 위치: 좌상단 (↶ ↷ 한 쌍)
- 기존 DrawingCanvas 의 Undo 스택(20단계)에 Redo 추가
- Cmd+Z / Cmd+Shift+Z 단축키 함께 지원 (예정)

## 8. 시계 (PM 시간 표시)

- 위치: 우상단 (예: `PM 2:30` 형태)
- 게임 내 시간과 동기화 (정보 표시 전용)
- 손님 인내심 카운트다운은 **표시하지 않음** (이번 단계에선 시간만 흐름)

## 9. SUBMIT / RESET

- SUBMIT: 하단 좌. 누르면 평가 단계로 진입
- RESET: 하단 우. 캔버스 초기화
- 서명·제목 입력 인터랙션은 **후순위** (지금 단계 미포함)

## 10. 활성 도구 시각 표시

- 선택된 도구만 **살짝 위로 뜨는 정도**
- 글로우/외곽선/그림자 강조 사용 안 함
- 호버 시: 가벼운 scale up 또는 살짝 들리는 Tween (정밀 수치는 구현 단계에서)

## 11. 후순위 / 미정

- 추가 브러시 도구 해금 (스프레이, 마커, 스머지 등)
- 추가 팔레트 색상 해금
- 서명 / 제목 입력 인터랙션
- 굵기 슬라이더 디자인
- 컬러피커 팝업의 정확한 위치 / 닫기 인터랙션
- 스포이드 활성 시 커서 모양 변경 여부

## 참고 자료 (리서치 노트)

> 초기 조사 내용. 위 확정안의 근거.

그림을 그리는 UI를 설계할 때, 이미 출시된 그림 게임 / 디지털 페인팅 앱이 제공하는 도구와 유저들이 요구하는 기능을 참고하면 도움이 됩니다. 관련 게임 및 커뮤니티에서 공통적으로 등장하는 요소를 조사해보았습니다.

### 그림 게임·페인팅 앱에서 공통적으로 제공하는 기능

| 범주 | 근거 | 설계 시 고려 사항 |
| --- | --- | --- |
| 붓/도구 선택 | 패스파르투1은 단순히 "여러 가지 색과 크기별 붓"만 제공하고 기본적으로 에어브러시는 잠금 해제해야 합니다. 패스파르투2는 도구마다 텍스처와 투명도, 흐름, 스탬프 패턴을 달리해 색다른 감촉을 주도록 설계했으며, 물감의 "습기" 같은 물성도 구현해 붓마다 개성이 있도록 했습니다. | 플레이어가 사용할 붓 종류(펜, 두꺼운 붓, 스프레이, 에어브러시 등)를 선택할 수 있도록 해야 합니다. 각 붓은 굵기와 질감이 달라야 하고, 게임 진행에 따라 새로운 붓을 얻도록 하면 동기부여가 됩니다. |
| 붓 굵기·불투명도 조절 | Procreate에서는 화면 옆에 브러시 크기 슬라이더와 불투명도 슬라이더가 있어 굵기와 투명도를 즉시 조절하고, 패스파르투 시리즈 역시 마우스 휠로 붓 크기를 조절합니다(커뮤니티 언급). | UI에 별도의 슬라이더를 두어 브러시 크기와 투명도를 조절할 수 있게 합니다. 붓 아이콘에 마우스를 올리면 현재 굵기를 미리 볼 수 있는 미리보기도 유용합니다. |
| 색상 선택·팔레트 | 패스파르투1은 24개의 색상만 선택할 수 있습니다. Procreate에서는 화면 우측 상단의 색상 원을 누르면 컬러 패널이 열리고, 색상 원반에서 색조와 명암을 선택하며, 팔레트 라이브러리에서 자신만의 팔레트를 만들거나 가져올 수 있습니다. | 기본 팔레트와 함께 색상 원반을 제공해 사용자가 직접 색을 선택할 수 있게 하고, 즐겨 쓰는 색을 저장하는 팔레트 기능을 제공합니다. 일부 게임처럼 초반에는 제한된 색만 주고 스토리 진행에 따라 새 색상이 해금되는 방식도 재미요소가 될 수 있습니다. |
| 지우개/되돌리기 | 디지털 드로잉 웹앱 중 하나는 실시간으로 작업을 저장하면서 Undo/Erase/붓 크기/색상 변경 기능을 제공해 초등 학생도 쉽게 사용할 수 있다고 소개합니다. Procreate에는 브러시 슬라이더 아래에 되돌리기/다시 실행 화살표가 있으며, 에러를 지울 때는 상단의 지우개 아이콘을 선택해 원래 브러시처럼 크기와 모양을 조절할 수 있습니다. | 캔버스 전체를 초기화하는 리셋 버튼, 한 단계씩 작업을 취소/다시 실행하는 Undo/Redo 버튼, 브러시 크기와 동일한 방식으로 조절 가능한 지우개 도구를 제공합니다. 지우개 역시 여러 모양을 선택할 수 있게 하면 세밀한 수정이 가능합니다. |
| 색상 추출(스포이드) | Procreate는 크기·불투명도 슬라이더 사이에 스포이드(eyedropper) 버튼을 배치해 화면의 특정 색을 바로 추출할 수 있게 합니다. | 캔버스에서 원하는 색을 빠르게 추출하도록 스포이드 기능을 배치합니다. |
| 완성/제출 | 패스파르투에서는 그림을 완성한 뒤 이름을 붙이고 전시할 수 있습니다. | '제출' 버튼을 누르면 게임의 평가 단계로 넘어가도록 하고, 그림에 제목을 붙이거나 서명을 하는 인터랙션을 추가하면 몰입도가 높아집니다. |
| 추가 도구 | Procreate의 Magic Wand 메뉴에는 색조·밝기 조절, 블러, 왜곡 등 다양한 후처리 도구가 있습니다. 또 QuickShape 기능은 선이나 원을 완벽한 형태로 보정합니다. | 단순한 캐리커처 게임에서는 모든 후처리 기능이 필요하지 않을 수 있지만, 버킷 채우기, 대칭/미러 도구, 완벽한 원이나 직선을 그리는 QuickShape, 스머지(번지기) 같은 기능을 일부 추가하면 그림 표현력이 커집니다. |
| 조작 편의성 | Chicory: A Colorful Tale는 콘솔에서 오른쪽 스틱으로 붓을 움직이고 어깨 버튼으로 붓의 크기와 색을 즉시 변경할 수 있어 조작이 매우 간편하다고 평가했습니다. | 키보드/마우스뿐만 아니라 패드나 터치 스크린 지원 시에도 빠르게 도구를 바꿀 수 있는 단축키를 제공합니다. 마우스 커서를 아이콘에 올리면 아이콘이 살짝 확대되는 호버 효과를 추가해 계획한 연출을 구현합니다. |

### 유저들이 원하는 기능과 UI 개선점

- **다양한 붓과 개성 있는 표현**: 패스파르투2의 개발팀은 "각 도구가 고유한 질감과 흐름을 갖도록 설계했다"고 밝혔습니다. 많은 플레이어가 단순한 붓 몇 개만 있는 Passpartout 1의 제한적인 표현에 아쉬움을 표했으며, 브러시 종류가 늘어나고 물성까지 반영되는 Passpartout 2에 긍정적입니다.
- **편리한 사이즈·색상 조절**: 브러시 크기와 색상을 빨리 바꾸고 싶다는 의견이 많습니다. Procreate처럼 슬라이더와 스포이드, 퀵 컬러 전환 기능을 제공하면 조작이 간단해집니다.
- **Undo/Redo와 편리한 지우개**: 실수했을 때 되돌리기와 지우개가 필수라는 의견이 많습니다. Procreate와 여러 웹앱에서 이를 기본 제공하며, 게임에서도 20단계 Undo와 지우개 도구를 구현해두신 만큼 UI에 잘 드러내야 합니다.
- **색상 팔레트와 제한된 색 해금**: 일부 게임은 일정 수의 색상만 제공하고 진행에 따라 추가 색상을 해금하는 구조입니다. 이러한 제한은 캐리커처 게임의 난이도와 성장감을 조절하는 요소가 될 수 있습니다.
- **그림 제출 및 이름 짓기**: Passpartout에서는 그림을 완성한 뒤 이름을 붙이고 전시하는 과정이 있어 애착을 높입니다. 캐리커처 게임에서도 손님에게 작품을 보여주고 반응을 받는 흐름과 함께 제목/서명 기능을 넣어보세요.

### 종합 제안 (초기안)

> 초기 리서치 단계의 일반론. 최종 확정안은 위 1~11번 섹션 참조.

1. **UI 구성**
    - 좌측에 캔버스 영역을 크게 두고 우측에는 툴바 영역을 배치합니다.
    - 툴바에는 브러시 선택 메뉴, 색상·팔레트 패널, 브러시 크기/불투명도 슬라이더, 지우개, 스포이드, 버킷 채우기, Undo/Redo, 초기화, 제출 버튼을 배치합니다. 호버 시 아이콘이 확대되어 사용자에게 피드백을 줍니다.
2. **기능 단계별 해금**
    - 초반에는 단순한 연필과 한정된 색상으로 시작하고, 진행에 따라 에어브러시·스프레이·펜·스머지 등 새 도구와 색상 팔레트를 해금합니다.
3. **조작 편의성**
    - 브러시 크기·색상 변경을 키보드 단축키나 휠 스크롤로 빠르게 처리할 수 있게 합니다. 콘솔 지원 시에는 스틱+버튼 조작을 적용하여 Chicory처럼 직관적인 조작을 구현합니다.
4. **그림 제출 흐름**
    - 제출 버튼을 누르면 작품을 저장하고 손님에게 보여주며 점수와 대사를 출력합니다. 필요한 경우 제목을 입력하게 하여 플레이어의 참여를 유도합니다.
