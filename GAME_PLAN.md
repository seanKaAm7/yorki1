# Yorki, the Portraitist — 게임 기획서

> 버전 0.1 | 작업 중

관광지를 돌아다니며 사람들의 캐리커처를 그려주고, 조용한 하루들을 보내는 동안 성장과 이야기가 천천히 쌓여가는 감성 시뮬레이션 게임.

---

## 핵심 감각

이 게임의 가장 중요한 목표는 조용하고 좋은 브금을 들으면서, 나긋하게 손님들의 그림을 그려주는 경험이다.
플레이어는 그렇게 하루하루를 보내다가, 어느새 시간이 흐르고, 성장하고, 환경이 바뀌고, 스토리가 진행되는 흐름을 느끼게 된다.

이 게임의 중심은 빡센 운영이나 경쟁이 아니라 **잔잔한 몰입감과 자연스러운 축적감**이다.

---

## 플레이어 경험 목표

플레이어는 다음 감각을 느껴야 한다.

1. **조용한 집중감** — 좋은 브금과 함께 손님 얼굴을 바라보며 그림을 그리는 시간 자체가 편안해야 한다.
2. **작은 기대감의 반복** — 다음 손님은 어떤 사람일지, 어떤 반응을 할지, 오늘 하루는 어떤 분위기일지 기대하게 만들어야 한다.
3. **자연스러운 성장감** — 큰 이벤트가 아니어도, 매일 조금씩 실력과 환경과 평판이 달라지는 감각이 쌓여야 한다.
4. **뒤늦게 깊어지는 감정선** — 처음엔 힐링처럼 시작하지만, 시간이 지나면서 Yorki의 과거와 내면, 관계의 결이 서서히 드러나야 한다.

---

## 컨셉

관광지를 돌아다니며 사람들의 캐리커처를 그려주는 화가 시뮬레이션 게임.

- **레퍼런스:** Passpartout: The Starving Artist + 타이쿤
- **분위기:** 힐링 + 성장 + 약간의 긴장감
- **플랫폼:** PC (Steam) 우선 개발 → 이후 모바일 (App Store / Google Play)
- **그래픽:** 3D (최종 목표), 초기 프로토타입은 2D로 시작 가능
- **전체 방향성:** 따뜻한 성장 시뮬레이션. 초반엔 플레이어가 주인공의 숨겨진 과거를 모른 채 힐링처럼 진행하다가, 시간이 지나며 과거와 내면이 하나씩 드러남. 후반에는 다소 어두운 시기와 감정선이 들어오지만 서사 흐름 안에서 자연스럽게 녹아드는 구조.

## 주인공

**Yorki (29)** — 수입에 신경 안 쓰려 하지만 막상 돈이 생기면 좋아하는, 평범한 면도 있는 캐리커처 화가.  
Yorki만의 뚜렷한 특성은 캐릭터 작업 때 구체화 예정.
- 기본적으로 조용히 그림만 그림. 손님이 먼저 말을 걸면 잘 받아줌.

- 외형: 갸름한 얼굴, 약간 날카롭지만 부드러운 눈, 피곤하지만 차분한 표정. 밝은 갈색~다크 블론드 텁수룩한 머리. 마른 체형. 브라운/올리브 계열 후드 + 체크 셔츠 레이어. 과장되지 않은 현실적 디자인. 따뜻하지만 약간 멜랑콜리한 인디 게임 분위기. 비주얼 레퍼런스: Ryan Robinson (유튜버)
- **스프라이트 초안 완료:** 8방향 정지 + 8방향 걷기 애니메이션 (6프레임, Low Top-Down, 48px) — PixelLab 생성
- 출신: 독일 소외된 작은 마을 (홈타운)
- 베를린에서 3~4년 자취하며 회사 다님. 퇴사 후 고향으로 귀향.
- 부모님 둘 다 없음. 고향 빈 집에 혼자 돌아온 것.
- 세계관: 현실 기반 + 판타지 살짝 (캐릭터 작업 때 구체화)

### 배경 스토리
평범하게 회사 다니며 살아온 사람. 과거는 처음부터 드러나지 않음. 커리어가 쌓일수록 조금씩 밝혀지는 방식.

**프롤로그 (플래시백으로 야금야금 풀림):**
회사 생활 한 50% 쯤 됐을 때, 자기보다 한 단계 위 직급으로 여자 A가 들어옴. 그녀를 그려보고 싶다는 생각에 그림을 그리기 시작함. 실력을 쌓으며 "이제 줘도 되겠다" 싶었던 70~80% 시점에, 그녀에게 애인이 생김. Yorki는 그 뒤로도 사람들 사진을 보며 계속 그림을 그리고, 회사 일은 점점 복잡해지고 안 좋은 일이 연속으로 생김. 점점 스트레스가 쌓이다가 — 사람들을 왕창 그리고 싶다는 생각에 퇴사하고 길거리로 나옴.

플레이 중 이쁜 손님 그리다가 문득 그 기억이 떠오르는 플래시백 방식으로 야금야금 풀림.

**핵심 떡밥:** 그림을 시작한 이유는 좋아하던 여자에게 그려주려고. 그 여자가 어떻게 됐는지가 스토리 핵심.

즉, 그림은 단순한 취미가 아니라 Yorki의 상실과 그리움, 회피와 집착이 얽혀 있는 행위다.

### 주요 인물
- **Dazzy (여자 A)** — Yorki가 그림을 시작하게 된 원인. 베를린 회사 동료, 한 직급 위. Yorki가 먼저 본 게 아니라 그녀가 와서 얘기하는 걸 Yorki가 보면서 시작됨. 후반에 우연히 Yorki와 마주침. 외모 뛰어남, 성격 착함.
  - **옵션 1:** 여행 or 출장 중 Yorki를 발견하고 손님으로 등장.
  - **옵션 2:** Yorki가 베를린 스테이지에서 길거리 그림 그리다가 감회 새롭게 우연히 마주침.
- **Winter (여자 B)** — 코헴 거주. 4~5번째 손님으로 등장. 이후 지속적으로 마주침. 털털한 성격, 단발. Yorki랑 통하는 부분 있음. 중간에 한 번 싸움. 비주얼 레퍼런스: 레제 / 미카사 / 사이버펑크 / 인비저블 갤 스타일 — 시크하고 힘있는 인상.
- **Bon clay** — 중간에 합류해서 Yorki 옆에서 같이 그리는 친구. 봉클레가 그리는 그림 자체는 화면에 보여주지 않음. 진척도/성과 정도만 표시. 실력은 플레이어보다 약간 못한 수준으로 설정 — 비교 대상이자 동료. 봉클레와의 고유한 서사 예정. 합류 시점 및 방식 미확정.
- **고로 (튜토리얼)** — 초반에 신입을 본 고인물처럼 대해줌. 이것저것 알려주는 튜토리얼 역할.
- **할머니** — 홈타운 거주. Yorki 귀향 후 재회. Yorki가 어떤 상황이 돼도 응원하는 사람. 게임 내 등장 여부 미확정.

**로맨스는 메인스토리 아님.** 그림 그리고 성장하는 게 메인. A랑 B는 자연스럽게 엮이는 서브스토리.  
**루트 결정 방식:** 게임 전체에 걸쳐 대화/행동 선택마다 A포인트/B포인트가 보이지 않게 쌓임. 플레이어한테 수치 안 보여줌 → 자연스럽게 플레이하다 80% 시점에 높은 쪽으로 루트 확정. (레퍼런스: Dispatch의 보이지 않는 누적 카운터 방식)

**A/B 구도 중요 원칙:**
- B와 자주 만난다고 해서 무조건 B로 기울어지는 구조가 아님
- A도 충분히 현재형 선택지로 작동해야 함 — 대화도 하고 서사적으로 플레이어가 충분히 선택할 수 있는 인물이어야 함
- 관계는 단순 출현 빈도보다 플레이어가 느끼는 감정과 선택이 더 중요

## 손님 유형

손님 정체는 처음엔 모르고, 대화 중 힌트로 유추하는 방식.  
지역별로 등장하는 사람 다름 (스테이지마다).

### 특별 게스트 (스토리 연결)
- **CEO (모티브: 정의선)** — 제시를 함 (작품을 크게 산다든지, 기업에 넣게 해주겠다든지). 나중에 Yorki 인생에 엮일 수도 있음.
- **전 여친** — 뜬금없이 손님으로 등장. Yorki 과거 드러내는 장치.
- **휴직 중인 회사 동료/상사** — "왜 갑자기 나왔어요?" 같은 대화로 배경 스토리 자연스럽게 풀림.
- 스토리 상당히 진행 후: 유명인, 유명 정치인, 고위급 인사들한테 컨택이 옴.

### 빌런
- **그림 받고 튀는 애** — 나이 어림 (10살쯤). 아주 낮은 확률로 등장.
- **그림덕** — 그리는 내내 자꾸 말 검.
- **훈수충** — "아 좀 연필이 안 좋은 거 같은데.." 식으로 계속 훈수.
- **꾸민 성매매자** — 작업을 시작함. 새벽까지 일할 시 등장.

### 일반 손님
- **노숙자** — 가끔 배경에서 보이기도 하고 걷다가 쳐다봄.
- **어린 아이** — 엄마랑 둘이서 오거나 혼자 와서 그려달라고 함. 예의 바름.
- **노인** — 대부분 조용함.
- **학생** — 여고생들 다 같이 와서 잘 그린다고 해줌. 남학생들도.
- **직장인** — 20대 후반~30대 초 젊은 직장인들.
- **경찰** — 허가 받고 운영하는지 확인 (타시기처럼).
- **군인** — 3명씩 다니다가 들름.
- **일반인** — 성인 남성, 성인 여성.
- **래퍼** — 전형적인 래퍼 의상보다 디자이너처럼 입음. 성격 까칠하고 안 좋아 보이는데 생각보다 착함.
- **단골 손님** — 다른 손님도 데려다 줌.

## 핵심 시스템

### 이동 (방향 수정)
- 캐릭터를 직접 조종해 맵을 돌아다니는 방식 → **제거**
- 핵심 경험은 자리에 앉아 손님과 그림을 그리는 것. 이동은 수단이었고 목적이 아니었음.
- **Coffee Talk 방식으로 전환**: 배경 고정 + 손님이 찾아오는 구조

### 자리 선택
- 스테이지 내 여러 자리 중 하나를 선택 (맵뷰 일러스트에서 클릭)
- 선택하면 해당 배경으로 전환, 그 자리에서 손님을 맞이함
- 자리마다 분위기와 손님 유형이 다름 (좋은 자리 = 손님 많음)
- 초반: 불법 노점. 경찰/단속원이 손님으로 찾아와 대화로 쫓겨나는 연출 → 다른 자리 선택
- 중반: 정식 stall 구매 가능 → 쫓겨날 걱정 없음
- 후반: 더 좋은 위치, 더 큰 stall 업그레이드
- **레퍼런스: Coffee Talk** — 고정 배경 + 퀄리티 높은 아트로 몰입감 구현

### 그리기
- 마우스/터치로 직접 캐리커처 그림
- 판단 기준: 손님마다 내부적으로 얼굴 격자 기준점 존재 → 플레이어가 그린 선이 그 기준에 얼마나 맞는지로 점수 산정 (손님과 얼마나 비슷하게 그렸는지)
- **구현 완료 (프로토타입):** 8x8 코스 그리드 + 슬라이딩 윈도우(±2칸) 방식으로 픽셀 유사도 계산 (ScoreCalculator.cs)
- **구현 완료 (프로토타입):** Undo 기능 — Cmd+Z (macOS) / Ctrl+Z (Windows), 최대 20단계
- 보조 기능: 격자/점선 오버레이 (설정에서 켜고 끄기 가능 — 켜면 쉽고 끄면 어렵지만 판단 기준은 동일)
- 기본 타임: 10분
- 손님마다 인내심 다름 (여유로운 손님 = 넉넉하거나 제한 없음 / 바쁜 손님 = 1~3분)
- 다 그렸으면 시간 전에 미리 제출 가능
- 특별 손님(대기업 회장 등)은 갑자기 짧은 시간 제약 걸림 → 긴장감 포인트

#### 그리기 화면 UI 레이아웃

- **레퍼런스 초안 이미지:** `play ref/초안.png` (확정 기준)
- **레이아웃:** 1인칭 시점. 화면 좌측 손님(정면), 우측 이젤+캔버스+도구.
- **우측 구성:** 캔버스(이젤에 테이프로 고정된 스케치북 느낌), 물감 팔레트(동그란 색상 선택), 도구들 자연스럽게 배치, SUBMIT/RESET 버튼.
- **하단 대사창:** 손님 대사 표시.
- **시간 표시 (우측 상단):** PM 형식 (예: PM 2:30). 게임 내 하루 흐름과 연동.
- **분위기:** 실제 화가 책상처럼 도구들이 자연스럽게 놓인 느낌. UI보다 세계관에 녹아있는 방향.

### 손님 반응 연출
그림 제출 후 두근두근 줌인 → 결과 발표.

| 단계 | 반응 | 점수 기준 (프로토타입) |
|------|------|----------------------|
| 매우만족 (VerySatisfied) | 대사 + 수입 표시 | 87점 이상 |
| 만족 (Satisfied) | 대사 + 수입 표시 | 기준 미달 시 단계별 분기 |
| 애매 (Neutral) | 대사 | — |
| 불만 (Unsatisfied) | 대사 | — |
| 진상 (VeryUnsatisfied) | 대사 + 멘탈 수치 깎임 | — |

- **구현 완료 (프로토타입):** 5단계 반응 시스템 (ReactionSystem.cs, ReactionUI.cs)
- **구현 완료 (프로토타입):** 반응 대사는 하단 고정 대사창(DialogueUI)에 출력, 3초 후 캔버스 초기화 후 내레이터 화면으로 복귀
- 대사는 미리 대량 생성 후 로컬 저장 (실시간 API 호출 X → 비용 문제 없음)
- 조건 태그 조합으로 적절한 대사 출력: [손님유형] + [퀄리티 단계] + [특징] + [시간대] + [친밀도] 등
- 조합별로 대사 풀 구성, 그 안에서 랜덤 출력 → 반복감 최소화
- 손님마다 원하는 스타일, 인내심, 지불 금액 다름

### 대사 시스템 구조

게임 진행 상태에 따라 대사가 자연스럽게 달라지는 구조. 조건문을 직접 늘리는 게 아니라 **태그 필터링** 방식으로 관리.

**대사 데이터 구조 (CustomerData 확장):**
- `greetingLines` — 손님 등장 시 대사
- `chattingLines` — 그리는 중 랜덤 대사
- `satisfiedLines` — 잘 그렸을 때 반응 대사
- `unsatisfiedLines` — 못 그렸을 때 반응 대사

**태그 조건 (필터링 기준):**
- 방문 횟수 (첫 방문 / 단골)
- 시간대 (낮 / 저녁 / 밤)
- 친밀도 수치
- 스테이지 (코헴 / 베를린 / 등)
- 플레이어 실력 단계

**작동 방식:**
GameManager가 현재 게임 상태(시간대, 친밀도, 방문 횟수 등)를 보유. 대사 출력 시점에 조건에 맞는 대사 풀에서 랜덤 1개 선택. 대사 추가는 코드 수정 없이 데이터 파일(asset)만 편집하면 됨.

### Yorki 말 안 듣는 연출

- 평소엔 플레이어 조작에 순순히 따름
- 특정 사건 발생 1~2일 전부터 퇴근 시도 시 Yorki가 거부하고 계속 일함 ("아직 한 명만 더" 등)

- 플레이어가 조종하면서 자연스럽게 "얘 왜이래?" 느끼게 되는 연출

## 스토리 흐름

**프롤로그**
베를린 회사 다니던 Yorki, 여자 A 보고 그림 시작. A한테 애인 생긴 걸 SNS로 알고 → 잊으려고 다른 사람들 계속 그림. 스트레스 쌓이다 퇴사, 고향으로 귀향.

**초반 (홈타운 → 코헴)**
길거리 나가서 처음 장사 시작. 튜토리얼 고로. 코헴에서 B 처음 만남. 스테이지 간 씬으로 과거 조각조각 풀림.

**중반**
B랑 지속적으로 마주치며 관계 쌓임. 대화/행동 선택들로 A포인트/B포인트 무의식 중에 쌓임. B랑 한 번 싸움.

**80% 시점 (슬럼프)**
Yorki 그림 집착 심해짐. 특정 사건 1~2일 전부터 플레이어가 퇴근시키려 해도 말 안 들음. 어릴 때 부모님 그리던 과거 스근하게 나옴. A 등장 (손님으로 or 베를린 스테이지에서 마주침). 쌓인 포인트에 따라 루트 확정. 선택한 사람 덕분에 슬럼프에서 나옴.

**엔딩 분기**
- 화가로 실패 → 복직 엔딩
- A 루트 → 미확정
- B 루트 → 미확정
- 중간 엔딩들 (미확정)

## 스테이지 간 씬

도시에서 도시로 이동할 때마다 짧은 회상 씬이 나옴.  
검은 배경 + 텍스트만. 애니메이션 없음.  
Yorki의 과거 대화가 흘러나오는데, 말하는 사람이 누군지 표시되지 않음.  
플레이어가 대화를 보면서 퍼즐 맞추듯 맥락을 유추하는 방식.  
레퍼런스: Super Paper Mario — Blumiere & Timpani 챕터 간 씬

단순 삽입이 아니라 **감정적으로 집중 가능한 연출**이어야 함. 플레이어가 제대로 몰입해서 볼 수 있게 만드는 게 중요.

**구조적 장치:**  
프롤로그에서 플레이어는 "여자 A를 보고 그림을 시작했다"는 표면적인 이유만 알게 됨.  
스테이지 간 씬이 쌓이면서 사실은 어릴 때부터 훨씬 깊은 계기가 있었다는 게 드러남.  
프롤로그에서 나온 내용과 같은 시간대여도, 씬에서는 그 사이의 세밀한 감정과 몰랐던 디테일을 암시하는 새로운 대화가 나와야 함.

예시 대사: "오랜만에 그리려니까 어렵네…"  
→ 플레이어는 여자 A를 보고 처음 그린 줄 알지만, 이 대사로 전에도 그린 적 있었다는 게 암시됨. 어릴 때 미친 듯이 그렸던 시기는 아직 안 나왔기 때문에 의문만 생기는 구조.

대화 순서는 과거→현재 순서 안 지켜도 됨. 뒤죽박죽 섞이거나 과거로 돌아가도 됨. 플레이어가 퍼즐 맞추듯 유추하는 방식.

여자 A의 SNS 프로필에서 애인 생긴 걸 알게 됨 → 그녀를 위해 그리던 그림을 접고, 잊으려고 다른 사람들을 계속 그려주기 시작함.

## 개발 방향

- 그래픽/비주얼 스타일 확정 전에 **기능 위주 프로토타입 먼저** 제작
- 프로토타입 단계에서 비주얼은 무료 에셋 or 임시 박스로 대체 가능
- 핵심 검증 목표: 그리기 캔버스 + 손님 반응 시스템이 실제로 재밌는지
- 그래픽 방향: **도트 (확정)**
- 비주얼 레퍼런스 (각 요소별):
  - **Coffee Talk** — 플레이 화면 레이아웃 및 구도. 1인칭 시점, 손님 좌측 정면, 우측 UI 패널.
  - **황혼의 공주 (젤다의 전설)** — 색감 및 분위기. 어둡고 무거운 갈색/금색 톤, 음울하면서 감성적인 분위기.
  - **Hyper Light Drifter** — 픽셀아트 스타일. 순수 2D, 색감 강한 픽셀아트.
  - 세 가지를 조합해 Yorki만의 색깔 구축 예정. 레퍼런스 추가 수집 중.
- Octopath Traveler HD-2D 방식 → 제거 (순수 2D 픽셀아트로 확정)

### 씬 구조

#### 전체 씬 목록
- **Scene A — 캔버스:** 그리기 화면. Coffee Talk 방식 — 1인칭 시점, 좌측 손님 정면, 우측 이젤+캔버스+도구. 레퍼런스: `play ref/초안.png`.
- **Scene B — 자리/대기 화면:** 캐릭터 직접 이동 없음. 자리 선택 후 고정 배경에서 손님을 기다리는 구조. 손님이 찾아오면 Scene A로 전환. 평가/반응 후 Scene B로 복귀.
- **World Map — 세계 맵:** 스테이지(도시) 간 이동 화면. 인지도 상승에 따라 이동 가능 지역 열림.
- **Menu / Setting / Dialog:** UI 씬들.

#### 프로토타입 현재 구조 (단일 씬, 추후 분리 예정)
- GameState(Narrator / Drawing) 열거형으로 UI 토글 방식 운영
- Narrator 상태: 검정 화면 + 하단 대사창. 엔터/스페이스로 대사 진행
- Drawing 상태: 캔버스 + 손님 이미지 + 툴바 표시
- 대사창(DialogueUI)은 항상 하단에 고정 — 내레이터 대사, 손님 반응 대사 공용으로 사용

## 노트

후에 나오는 내용들론 언제부터 그렇게 그림에 집착하게 됐는지가 나온다.

어렸을 때 요키가 — 부모님이 어릴 때 둘 다 돌아가심. 할머니 밑에서 잘 자랐지만 마음속엔 구멍이 있었음. 그리움에 계속 그림으로 부모님을 그렸던 것.

10대 후반~20대 초반에 전 여친과 지내면서 깨달음을 얻음 — 누군가를 그리워하며 계속 그려내고 머물러있는 게 답이 아니라는 것. 그래서 자연스럽게 그림을 내려놓음. 전 여친이 직접 뭘 한 게 아니라 Yorki 스스로 깨달은 것. 헤어진 이유 미확정.
- **옵션:** Yorki가 특정 무언가에 obsess하는 성향이 있었고 그게 원인일 수도 있음.

베를린에서 여자 A를 보고 본능적으로 "그리고 싶다"는 감각이 돌아옴. 의도한 게 아니고 거부감도 없었음. 그냥 자연스럽게 다시 시작된 것.
여자 A한테 애인이 생긴 걸 SNS로 알고 난 후 — 잊으려는 회피 수단으로 다른 사람들을 계속 그리기 시작함. 전엔 그리움 때문에 그렸고, 이번엔 잊으려고 그리는 것. 동기는 다르지만 패턴은 같음.

그래서 애써 그림을 기억 저 편에 묻어놨던 것임. 전 여친과 헤어진 후 베를린으로 가 회사를 다님. 그러나 회사에서 여자 A를 보며 순간적으로 그려야겠다는 생각이 들었음.

요키는 점점 스토리가 진행될수록 그림에 집착함.

스토리 80% 쯤에 플레이어들이 Yorki를 보며 "얘 왜이래?" 하게 되는 시점이 옴. 특별한 사건이 터지는 게 아니라, 플레이하면서 Yorki가 그림에 점점 과하게 집착하는 모습이 쌓여서 자연스럽게 드러나는 것. 플레이어 본인도 조종하면서 느끼게 됨. 그 시점에 안 풀어줬던 어린 시절 과거가 스근하게 나옴.
