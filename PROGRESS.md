## 알려진 문제점

### 스프라이트 품질

| 문제 | 내용 | 심각도 |
|------|------|--------|
| 누끼 불완전 | 일부 스프라이트 배경 제거가 제대로 안 됨 | 높음 |
| 캐릭터 위치 불일치 | 스프라이트마다 캐릭터 위치가 달라 컷 전환 시 위아래로 튐 | 높음 |
| 캐릭터 크기 불일치 | 스프라이트마다 크기가 달라 전환 시 줄었다 늘었다 보임 | 높음 |
| 의자 다리 잘림 | 일부 컷에서 의자 하단이 잘려있음 | 중간 |
| 컷 전환 어색함 | idle↔talk 토글 시 몸 전체 자세가 달라 순간이동처럼 보임 | 높음 |

### 컷 전환 어색함 원인

idle↔talk 두 컷만 0.12초 간격으로 토글하는 구조인데, 두 컷의 몸통 자세 자체가 다르면 어색하다.
자연스럽게 하려면 두 컷은 **몸통 자세 동일 + 입 모양만 차이**여야 함.
현재 컷들은 PixelLab으로 생성 시 자세까지 달라진 것으로 추정.

### 해결 방향 — 스프라이트 정렬 기준 (미확정, 추후 결정 필요)

AI로 뽑은 컷은 매번 크기/위치가 미세하게 달라서 후처리 정렬이 필수.

**핵심 원칙 (방향만 확정)**
- 모든 컷을 같은 캔버스 크기 안에 넣고, 같은 기준점에 맞춘다
- idle/talk 쌍은 몸통 자세 동일 + 입 모양/손 위치만 차이
- 프롬프트만으로는 완벽하게 안 맞음 — 최종 정렬은 수동 필수

**정렬 작업 흐름 (예시 기준, 실제 수치는 미확정)**
1. 마스터 기준 컷 1개 선정 (neutral_idle 권장)
2. 기준 컷을 반투명으로 깔고 다른 컷을 올려서 맞춤
3. 맞추는 순서: 머리 크기 → 눈 위치 → 목 위치 → 어깨 폭 → 캔버스 기준점
4. 투명 PNG export → 게임에서 모든 컷 같은 좌표에 배치

**미확정 항목 (작업 전 결정 필요)**
- [ ] 캔버스 크기 (예: 1024×1024, 512×512 등)
- [ ] 기준점 위치 (예: 목 중심+어깨 라인, 허리 중앙 등)
- [ ] 캐릭터 표시 영역 (상반신만 / 허리 이상 등)

**작업 도구**
Aseprite, Photopea, 클립스튜디오 등 레이어 반투명 기능 활용
